# i18n  Internacjonalizacja

## Opis

Konfiguracja tłumaczeń oparta o `i18next` + `react-i18next`. Obsługuje języki: `en` (domyślny) i `pl`. Tłumaczenia ładowane są statycznie z plików JSON  brak potrzeby fetch'owania po stronie klienta.

## Pliki

| Ścieżka | Rola |
|---|---|
| `lib/i18n/config.ts` | Inicjalizacja i18next, lista zasobów |
| `components/providers/I18nProvider.tsx` | Client-side provider owijający drzewo React |
| `locales/en/common.json` | Tłumaczenia angielskie |
| `locales/pl/common.json` | Tłumaczenia polskie |

## Użycie w komponentach (Client Components)

```tsx
"use client";
import { useTranslation } from "react-i18next";

export default function Navbar() {
  const { t } = useTranslation();

  return <nav>{t("nav.home")}</nav>;
}
```

## Zmiana języka w runtime

```tsx
"use client";
import { useTranslation } from "react-i18next";

export default function LanguageSwitcher() {
  const { i18n } = useTranslation();

  return (
    <button onClick={() => i18n.changeLanguage(i18n.language === "en" ? "pl" : "en")}>
      {i18n.language === "en" ? "PL" : "EN"}
    </button>
  );
}
```

## Interpolacja zmiennych

```json
// locales/en/common.json
{ "welcome": "Hello, {{name}}!" }
```

```tsx
t("welcome", { name: "Kacper" }) // → "Hello, Kacper!"
```

## Dodawanie nowego języka

1. Utwórz `locales/<lang>/common.json` z tłumaczeniami.
2. Zaimportuj go w `lib/i18n/config.ts` i dodaj do obiektu `resources`.
3. Dodaj do tablicy `supportedLocales`.

## Dodawanie nowej przestrzeni nazw (namespace)

Domyślny namespace to `common`. Dla dużych sekcji (np. dashboard) możesz dodać osobny namespace:

1. Utwórz `locales/en/dashboard.json`.
2. Dodaj do `resources` w configu: `en: { common: enCommon, dashboard: enDashboard }`.
3. Użyj: `const { t } = useTranslation("dashboard")`.

## Struktura kluczy

Klucze są zagnieżdżone i opisują kontekst widoku:

```
nav.*         → nawigacja
hero.*        → sekcja hero
auth.*        → formularze logowania/rejestracji
errors.*      → komunikaty błędów
seo.*         → tytuły i opisy meta
```
